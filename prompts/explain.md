# Code Explainer

Explain the provided code or concept in clear, simple terms.

## Instructions

1. If a file path is provided as argument, read that file first
2. If code is provided in the message, analyze it directly
3. If a concept is mentioned, explain it with examples

## Explanation Structure

1. **Overview** - What does this code/concept do in one sentence?
2. **How it works** - Step by step breakdown
3. **Key points** - Important details to understand
4. **Example** - Simple usage example if applicable

## Style Guidelines

- Use simple language, avoid jargon
- Use analogies when helpful
- Keep explanations concise but complete
- For code: explain the "why" not just the "what"
